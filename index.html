VAE-GAN

(2020-2021)

During sixth form I read Generative Deep Learning by D. Foster which got me really excited about applying ML to generative tasks (specifically for images: it's amazing how we can use such rigid rules to produce organic shapes). After participating in Google CodeIn 2019 (for TensorFlow), I wanted to apply what I had learnt to a more difficult project, so I created my own implementation of a VAE-GAN.

The VAE-GAN framework trains a generator by combining the losses from a VAE and a GAN. The VAE loss focusses more on correctly reconstructing the content of an image, while the GAN loss is more focused on creating an image that has a realistic style, so combining the two (hopefully) gives us the advantages of both. I trained the model of my own dataset of faces scraped from IMDB to produce these results.


RL fencing agent

(2020-2021)

The way professionals play games such as chess or go has been revolutionised with the use of Reinforcement Learning (RL) agents that are able play at a substantially higher level than the best humans. After reading Reinforcement Learning by Sutton and Barto, I wanted to see if I could create a similar agent in the sport of fencing.

The difficulty with this is that fencing is a lot messier than board games: the state space is difficult to define, there are many more factors that could affect decisions, and decisions are made continuously (we don't take turns).

My idea was to manually classify each action in a bank of videos of myself fencing into a predefined action class and then train a RL agent that uses this information along with the current match state (score, time, action history, ...) to predict the value of each possible action at any point in the match, where the value of winning a match is 1.

A value function for a state action pair in fencing is actually very useful. You could use it to predict which fencer is more likely to win, which actions are most effective at each point, and what a fencer's strengths and weaknesses are. Analysis of the model's weights indicated that the actions which had the most impact on the value were a stop hit (lower value) and a fleche (higher value). The latter did not appear in the dataset, so we can discount this, but the former makes some sense because scoring a stop hit doesn't really give you as much to build from in your next hit (although it is possible this prediction is due to me going for stop hits when I'm out of ideas).

This project is something I would like to revisit: I think it would benefit from a more effective data collection strategy (I imagine some kind of pose estimation or learnt action encoding). Nonetheless, I really enjoyed this project because I think RL will be increasingly relevant in the future, as it teaches an agent to be intelligent rather than to mimic something that is intelligent.
